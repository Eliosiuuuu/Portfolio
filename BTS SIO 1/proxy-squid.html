Proxy Squid

Introduction et Contexte :

Les postes de travail de Projetlec sont désormais protégés. Le directeur administratif et financier souhaite renforcer la sécurité en régularisant les accès internet via un proxy. Ce serveur mandataire permettra de sécuriser le réseau, contrôler les accès et améliorer la gestion des connexions des utilisateurs.

Cahier des Charges :

  • Étude comparative : Analyse de l’intérêt d’un proxy par rapport à d’autres solutions d’accès à Internet
  • Installation : Mise en place du proxy Squid avec un POC (Proof of Concept) sur 3 postes clients
  • Intégration : Compatibilité avec un Active Directory pour gérer les utilisateurs
  • Analyse des performances : Comparaison avant/après pour mesurer les améliorations
  • Documentation : Notice d’utilisation pour lire les audits d’accès

Avantages et Inconvénients d’un Proxy :

Avantages :

  • Anonymat : Masque les adresses IP des utilisateurs
  • Filtrage : Contrôle des contenus et blocage des sites non autorisés
  • Performance : Mise en cache des données pour accélérer les requêtes
  • Sécurité : Protège contre les URL malveillantes et les attaques

Inconvénients :

  • Ralentissement : Augmente légèrement le délai des requêtes
  • Complexité : Configuration avancée nécessaire
  • Compatibilité : Certains sites ou applications peuvent poser problème

Infrastructure et Prérequis :

  • Serveur Linux avec 20 Go de stockage et 2 Go de RAM
  • Logiciel Squid Proxy Server et outils de monitoring

Plan de Mise en Œuvre :

Le projet a inclus :

  • - Installation et configuration du Proxy Squid
  • - Tests de performance et ajustements sur un POC
  • - Création d’un Domaine Active Directory
  • - Formation et documentation pour les utilisateurs

Conclusion :

Ce projet a permis de renforcer nos compétences techniques tout en mettant en lumière les enjeux d'intégration d'un proxy en entreprise. Bien que le projet n'ait pas été entièrement finalisé, les apprentissages en matière de sécurité réseau seront précieux pour de futurs projets.
